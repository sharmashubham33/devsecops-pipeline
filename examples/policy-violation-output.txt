This is what a developer sees when their Dockerfile violates policies.
The pipeline BLOCKS the deployment and the developer must fix these issues:

1. Pin the base image to a specific version tag
2. Add a USER instruction to run as non-root
3. Add required OCI labels for traceability

The error messages are specific enough that the developer knows exactly
what to fix without needing to read the policy source code.
